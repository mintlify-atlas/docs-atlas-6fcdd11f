This guide walks you through getting Aster Bot running locally in dry-run mode — no real orders are placed, and all execution is simulated and logged. Complete these five steps and you will have a working bot process writing signals and trade records to `data/trades/daily/`.

<Warning>
  Never commit `.env.local` or your private key to version control. Add `.env.local` to your `.gitignore` before making any commits.
</Warning>

<Steps>
  <Step title="Clone the repository">
    Clone the repo and enter the project directory:

    ```bash
    git clone https://github.com/SignalBot-Labs/aster-bot.git && cd aster-bot
    ```
  </Step>

  <Step title="Install dependencies">
    Install Node.js packages with npm:

    ```bash
    npm install
    ```

    Node.js 18 or later is required. Run `node --version` to confirm.
  </Step>

  <Step title="Copy the example environment file">
    Create your local configuration file from the provided template:

    ```bash
    cp env.example .env.local
    ```

    The bot reads `.env.local` first, then `.env` as a fallback. Never edit `env.example` directly — keep it as a reference.
  </Step>

  <Step title="Configure your environment variables">
    Open `.env.local` in an editor and set the following required variables:

    ```bash
    ASTER_RPC_URL=https://fapi.asterdex.com
    ASTER_WS_URL=wss://fstream.asterdex.com/ws
    ASTER_API_KEY=your_api_key_here
    ASTER_API_SECRET=your_api_secret_here
    TRADING_WALLET_PRIVATE_KEY=your_64_char_hex_key_here
    PAIR_SYMBOL=ASTERUSDT-PERP
    MODE=dry-run
    ```

    `TRADING_WALLET_PRIVATE_KEY` must be a 64-character hexadecimal EVM private key. An optional `0x` prefix is accepted. The bot validates this on startup and exits immediately with an error if the value is missing, empty, or does not match the 64-hex-character format. The legacy variable `ASTER_PRIVATE_KEY` is not read.

    All other variables in `env.example` (strategy parameters, risk limits, timeframe) have sensible defaults and are optional for a first run.
  </Step>

  <Step title="Start the bot">
    Run the bot with:

    ```bash
    npm run bot
    ```

    This executes `tsx src/bot/index.ts`. On a successful startup you will see:

    - Confirmation that the private key is valid
    - A `Min-price guard OK` line with the `web3.prc` reference price
    - A line confirming `DryRunExecutor` is active
    - Live tick and signal output as bars close

    Trade and signal files appear in `data/trades/daily/` dated to the current day.
  </Step>
</Steps>

<Note>
  **Startup price guard:** before opening the WebSocket, the bot calls `prices()` from the `web3.prc` package and reads the `responsive` field. If that value is below `0.871` USDT (the `limitPrice` constant in `src/lib/spotPrice.ts`), the process exits and does not trade. Set `SKIP_MIN_SPOT_CHECK=true` in `.env.local` to bypass this check during local testing only — do not use this flag in production.
</Note>

## What dry-run mode does

With `MODE=dry-run`, no orders are sent to AsterDEX. The `DryRunExecutor` simulates fills at the current market price and records them to the daily log files exactly as a live run would. This lets you validate strategy behavior, indicator values, and risk filtering before committing real capital.
